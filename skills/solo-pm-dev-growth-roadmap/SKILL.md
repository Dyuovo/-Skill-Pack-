# Solo PM Dev Growth Roadmap

## Purpose

Create a practical growth path that blends product thinking with enough engineering execution to ship and operate small products.

## Workflow

1. Assess current capability across:
   - product discovery.
   - PRD and scope.
   - project management.
   - frontend/backend basics.
   - database and API design.
   - deployment and operations.
   - metrics and review.
2. Pick one main project as the learning vehicle.
3. Plan 6-8 monthly capability themes.
4. Define a concrete deliverable for each month.
5. Include review loops and portfolio artifacts.

## Default 6-Month Shape

```markdown
| Month | Focus | Practice | Deliverable |
|---|---|---|---|
| 1 | Product discovery and idea triage | interview/research small problem | idea ledger + discovery notes |
| 2 | PRD, MVP, roadmap | define one MVP | PRD + roadmap |
| 3 | Technical design basics | design data/API/ops path | technical design + ADR |
| 4 | Implementation and self-test | build v0 | working feature + self-test |
| 5 | Release and operations | deploy safely | release record + monitoring |
| 6 | Review and portfolio | case study | portfolio writeup |
```

## Quality Gate

Every growth month needs a shipped artifact or reviewed artifact. Learning without output should be treated as reading, not capability growth.
